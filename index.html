Make It Snappy

With only three days to invent and implement this project in a group of two, we created a game that allows users to play by snapping their fingers and waving their hands to catch a floating bird in its nest.

Because we wanted to integrate both the webcam and the microphone, we faced a particular challenge with the getUserMedia API. We used the gest.js library to capture the waving gestures that moves the nest (which catches our bird) back and forth, but this library was not set up to also take audio input.

To solve this problem, we edited the gest library to integrate sound input, since we could not call getUserMedia multiple times without creating lots of bugs. We ended up with one call to the API, which was better from a user experience perspective as well (since the user is only being prompted to grant permission to the game to use the microphone and webcame once).

Though a little buggy (make sure your sound is on mute!), this game was really fun to make and taught me a lot about using the webcam and microphone in tandem. Additionally, we had fun figuring out how to build a game, use webkit animations and try out our hands at creating a theme. Enjoy!

CodeCast

This two and a half week project was a particular joy to work on. We started with a simple problem: scribbling down code while trying to understand a lesson from an instructor was a constant battle in the classroom, and we wanted to solve it. Additionally, the many videos we had watched on CodeSchool, youTube and similar platforms often left us frustrated as beginners, since we wanted to interact with the code directly to absorb the lesson.

To address these problems, we decided to make CodeCast, an interactive, real-time application for students and instructors of code. Additionally, we created a video replay of the lesson, complete with audio, that allows users to pause and interact with the code at any point during the lesson.

During this process, we faced many challenges, to integrating web sockets and http requests, to recording, sending and storing audio files, to synchronizing our audio and visual video playback.

Because we wanted to make a dynamic, interactive video, we decided to store snippets of text on keypresses, and play it back at the same rate that it was typed. As a result, synchronizing the granular audio files with the visual text was a particularly interesting challenge. By recursively rendering the text using time stamp deltas, we were able to solve this problem and create a smooth and synchronized video playback.

Currently, you can interact with the replay as you would with other video (rewind, fast-forward and pause function normally), but in future, we would like to make this more consistent and reliable. Additonally, while we have a chat of sorts over web sockets, which allows students to answer questions (instructors can also allow students permission to type at any point during the lecture, and then revoke that permission), we would like to improve this comment area to have specific threads that appear in the instructor dashboard, so that the instructor can easily address any outstanding questions from one page once the lecture is complete.

While this project demonstrated to me the quintessential programmer conundrum, meet your deadline or add all the features and code refactors you want, it was truly an engaging, dynamic project that helped me learn so much about code and my own abilities. Check it out at the link above or here on Github.
